This guide walks you through the core SwitchPalette workflow from a fresh start. By the end you will have a key remapped and a macro assigned and running.

<Note>
  You need a keyboard running Vial-compatible firmware. Standard keyboards will not be detected. See [Device Setup](/configuration/device-setup) if your keyboard is not appearing.
</Note>

## Remap a key

<Steps>
  <Step title="Connect your keyboard">
    Plug your Vial-compatible keyboard into a USB port on your Windows machine. No driver installation is required — SwitchPalette communicates over the standard HID interface.
  </Step>

  <Step title="Launch SwitchPalette">
    Run the application. SwitchPalette starts in the background and adds an icon to the system tray. The main window opens automatically on first launch.

    <Tip>
      After the first launch, you can reopen the main window by clicking the tray icon.
    </Tip>
  </Step>

  <Step title="Select your keyboard">
    The **Device Selection** page opens first. SwitchPalette queries connected HID devices and displays all detected Vial-compatible keyboards as cards.

    Click the card for your keyboard. SwitchPalette connects over USB and reads the current keymap from the device.

    <Note>
      If your keyboard is not listed, unplug and replug it, then refresh the page. Confirm the keyboard is running Vial firmware — see [Device Setup](/configuration/device-setup).
    </Note>
  </Step>

  <Step title="Open the Editor page">
    After selecting a device, the **Editor** page opens. The visual keyboard layout displays your current keymap. Each key shows its current assignment.

    The layout reflects the physical key positions and widths for your specific keyboard model.
  </Step>

  <Step title="Remap a key">
    Click any key on the visual layout. The key palette appears at the bottom of the screen, showing all available keycodes organized by category.

    Click the keycode you want to assign. The change is sent to the keyboard immediately over USB — no save step required. Switch between layers using the layer selector to configure alternative assignments.
  </Step>
</Steps>

## Create a macro

Macros in SwitchPalette are triggered by the M1–M12 keys. These correspond to Windows virtual key codes F13–F24, which SwitchPalette reserves exclusively for macro execution.

| Macro key | Virtual key code |
|---|---|
| M1 | `0x7C` |
| M2 | `0x7D` |
| M3 | `0x7E` |
| M4 | `0x7F` |
| M5 | `0x80` |
| M6 | `0x81` |
| M7 | `0x82` |
| M8 | `0x83` |
| M9 | `0x84` |
| M10 | `0x85` |
| M11 | `0x86` |
| M12 | `0x87` |

<Steps>
  <Step title="Assign an M-key to a physical key">
    In the **Editor** page, click the physical key you want to use as your macro trigger. In the key palette, navigate to the macro key section and select **M1** (or any M1–M12 key). The assignment is written to the keyboard immediately.
  </Step>

  <Step title="Open the macro settings">
    In the key palette, switch to the **Macro** tab. Click the **M1** slot to open the macro settings dialog for that key.
  </Step>

  <Step title="Configure the macro action">
    Choose a macro type and fill in its details:

    | Type | What it does |
    |---|---|
    | **Text** | Pastes a fixed string using the clipboard |
    | **Combo** | Sends a simultaneous key combination (e.g. Ctrl+Shift+T) |
    | **Open app** | Launches an executable at a specified path |
    | **Infrared** | Sends a stored IR code via Beam |
    | **AI text convert** | Rewrites selected text using a Gemini AI prompt |

    <Tip>
      Start with the **Text** type — enter a phrase you type frequently, like an email signature or code snippet, and assign it to M1.
    </Tip>
  </Step>

  <Step title="Trigger the macro">
    Press the physical key you assigned M1 to. SwitchPalette's background keyboard monitor intercepts the F13 keycode (`0x7C`) and executes the macro immediately.

    <Note>
      Macro monitoring only runs when SwitchPalette is active (not in debug mode). Run the release build with `flutter build windows` to enable background key monitoring.
    </Note>
  </Step>
</Steps>
